01 대원칙. 명시적 this 바인딩이 없는 한 항상 성립.
- 전역공간에서의 this는 전역객체
- 어떤 함수를 메서드로서 호출한 경우 this는 메서드 호출 주체. 
  - [객체명.메서드명]의 this값은 객체명
- 어떤 함수를 함수로서 호출한 경우 this는 전역객체를 참조. 
  - 메서드의 내부 함수도 동일. 화살표 함수 둥으로 우회 가능.
- 콜백함수의 제어권을 넘겨받는 함수에 따라 콜백 함수 내부의 this 지정됨.
  - 지정되지 않은 경우 this는 전역객체 참조.
- 생성자 함수의 this는 생성되는 개별 인스턴스를 참조.
  - var kitty = new Cat; // Cat의 this는 kitty가 됨.

02 명시적 this 바인딩
- call, apply 메서드는 this를 명시적으로 지정하면서 함수, 메서드를 즉시 실행.
- bind 메서드는 this(+ 함수에 넘길 인수 중 일부)를 지정하면서 새로운 함수 return.
- 요소를 순회하면서 콜백 함수를 반복 호출하는 내용의 일부 메서드는 별도의 인자로 this를 받기도 함.
