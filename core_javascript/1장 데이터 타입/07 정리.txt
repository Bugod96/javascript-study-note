01. 자바스크립트 데이터타입: 기본형 & 참조형
- 기본형은 불변값.
- 참조형은 가변값. 

02. 
변수: 변경 가능한 데이터가 담길 수 있는 공간.
식별자: 변수의 이름. 변수명.

03. 
변수 선언: 메모리의 빈 공간에 식별자 저장 + 값은 undefined 할당
변수에 기본형 데이터 할당시,
 별도의 빈 공간에 데이터 저장 + 해당 주소를 변수의 값에 할당.

04. 
변수에 참조형 데이터 할당시, 
- 참조형 데이터 내부의 각 property를 위한 변수 영역을 별도로 확보. 
- 해당 변수 영역들에 각각 property의 식별자를 저장.
  - 각 property에 대응되는 데이터는 별도의 빈 공간에 저장.
  - 해당 데이터의 주소는 각 property의 변수 값에 할당.
- 최초의 변수에는 해당 property 변수 영역들의 주소를 값으로 하는 데이터를 저장.

    변수 영역1(@1002): 식별자: obj1 / 값: @5001
    데이터 영역1(@5001): @7001, @7002 ~ 
    데이터 영역2(@5002): 1
    데이터 영역3(@5003): 'bbb'
    객체@5001의 변수(property) 영역1(@7001): 식별자:a / 값: @5002 
    객체@5001의 변수 영역2(@7002): 식별자:b / 값: @5003 

05. 불변 객체
참조형 데이터도 '깊은 복사'를 통해 내부 property들을 일일이 복사하면 불변성 확보 가능.
일반적으로 관련 라이브러리나 메서드 활용.

06. 
- undefined는 어떤 변수에 값이 존재하지 않음을 의미.
- null은 사용자가 명시적으로 없음을 표현하기 위해 대입하는 값.

(원칙) 비어있음을 명시적으로 나타내고 싶다면 오로지 null만 사용할 것.
- 사용자가 명시적으로 변수에 undefined를 할당하는 것은 지양할 것. 
=> undefined는 값을 대입하지 않은 변수에 접근하고자 할 때 JS가 반환해주는 값으로 간주할 것.
이유) 사용자가 직접 지정한 undefined와 자바스크립트 엔진이 반환한 undefined는 서로 다르게 동작할 수 있음.
