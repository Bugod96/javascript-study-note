[클래스]
자바스크립트는 프로토타입 기반 언어. 
클래스 문법[ES6]을 통해 흉내낼 뿐, 상속이라는 개념도 사실상 없음.

----------------------------------------------------
[클래스와 인스턴스의 개념 이해]

클래스: 공통적인 속성을 모아 정의한 것. 추상적인 상위의 개념.
인스턴스: 어떤 클래스의 속성을 지니는 실존하는 개체. 구체적인 하위의 개념.
-------
음식(과일의 superclass / 귤류의 super-superclass) : 먹을 수 있음
- 과일(음식의 subclass / 귤류의 superclass) : 먹을 수 있음 + 나무에서 열림
  - 배
  - 사과
  - 바나나
  - 귤류(과일의 subclass / 음식의 sub-subclass) : 먹을 수 있음 + 나무 + 껍질과 신맛
    - 감귤
    - 오렌지
-------
> 클래스는 하위로 갈수록 상위클래스의 속성을 상속받으면서 구체적인 속성들이 추가됨.
  아무리 하위여도 하위클래스는 여전히 클래스. 추상적인 개념에 불과함.
> 인스턴스('instance') : 클래스에서 정의한 모든 조건들을 충족하는 하나의 사례.
-------
현실: 인스턴스가 먼저 존재 => 구분짓기 위해 클래스에 대입. 
    : 클래스는 언제나 추상적인 개념.

프로그래밍: 클래스를 정의 => 하나의 인스턴스는 하나의 클래스만을 바탕으로 생성.
- 다중상속이 지원되는 언어에서는 하나의 인스턴스가 복수의 클래스에 속할 수 있음.
- 이 경우 인스턴스는 해당 클래스들의 속성을 각각 직계 상속받게 됨.
- 하지만 이 경우에도 인스턴스 생성 당시에는 오직 하나의 클래스만을 호출하여 생성.
- 클래스는 사용하기에 따라 추상적인 개념이거나 구체적인 개체가 될 수 있음.
